08 Database / History Design Improvements
=========================================

目的
----

将来、過去の予測・ストーリーラインと実際の価格推移を比較し、分析ロジックを改善できるようにする。

現状の主要テーブル:
- symbols
- financial_statements
- market_metrics
- analysis_runs
- score_snapshots
- llm_settings
- storyline_sets
- storyline_scenarios
- universe_members

不足しているもの:
- 実価格推移の観測
- signal outcome
- storyline outcome
- user decision
- review event
- data quality history
- LLM run detailsのversion/provenance


DB-1: price_observations
-----------------------

目的:
分析後の実価格推移を保存する。

カラム案:
- id uuid primary key
- symbol_id uuid references symbols(id)
- symbol text not null
- observed_at timestamptz not null
- timeframe text not null  -- daily, weekly, intradayなど
- open numeric
- high numeric
- low numeric
- close numeric
- adjusted_close numeric
- volume numeric
- source text not null
- source_as_of timestamptz
- raw_json jsonb
- created_at timestamptz default now()

index:
- (symbol_id, timeframe, observed_at)
- (symbol, timeframe, observed_at)

unique:
- (symbol_id, timeframe, observed_at, source)


DB-2: signal_outcomes
--------------------

目的:
score snapshotやsignalが、その後どうなったかを評価する。

カラム案:
- id uuid primary key
- score_snapshot_id uuid references score_snapshots(id)
- analysis_run_id uuid references analysis_runs(id)
- symbol_id uuid references symbols(id)
- horizon text not null  -- 1w, 1m, 3mなど
- entry_reference_price numeric
- price_at_horizon numeric
- max_favorable_excursion_pct numeric
- max_adverse_excursion_pct numeric
- return_pct numeric
- benchmark_return_pct numeric
- excess_return_pct numeric
- hit_target_base boolean
- hit_stop boolean
- rr_realized numeric
- outcome_label text
- evaluated_at timestamptz
- created_at timestamptz default now()

outcome_label例:
- target_hit
- stop_hit
- neutral
- underperformed
- data_unavailable

DB-5: user_decisions
-------------------

目的:
アプリが出した候補に対して、ユーザーがどう判断したかを保存する。

カラム案:
- id uuid primary key
- score_snapshot_id uuid references score_snapshots(id)
- storyline_set_id uuid references storyline_sets(id)
- symbol_id uuid references symbols(id)
- decision text not null
- decision_reason text
- planned_entry_price numeric
- planned_stop_price numeric
- position_size_note text
- created_at timestamptz default now()

決定値例:
- reviewed
- watch
- entered
- skipped
- rejected
- deferred

注意:
売買自動執行ではない。ユーザーの判断記録として扱う。

RLS方針
-------

初期:
- enable row level security
- service_role policyのみ

将来multi-user化:
- owner_user_id追加
- auth.uid() = owner_user_id policy追加


outcome評価時の注意
------------------

- lookahead biasを避ける。
- score_snapshot時点で存在しないデータを使わない。
- universe変更を記録する。
- stock splitやadjusted priceを扱う。
- benchmark returnを保存する。
- delisted/suspended銘柄を除外しない。
