P0-1: finalEntryScore の上振れ修正
----------------------------------

対象ファイル:
- src/lib/scoring/scoring-engine.ts
- src/lib/scoring/scoring.test.ts

現在の問題:
computeFinalEntryScore と computeFinalEntryScoreUncapped が symbolScore * 0.50 + 50 のような構造になっており、中立symbolScoreでもfinalEntryScoreが高くなる。

変更内容:
1. computeFinalEntryScoreUncappedを以下方針へ変更する。

  function computeFinalEntryScoreUncapped(symbolScore: number, input: ScoringInput): number {
    const marketAdjustment = marketScoreAdjustment(input.market);
    const sectorAdjustment = sectorScoreAdjustment(input.sector);
    return clamp(symbolScore + marketAdjustment + sectorAdjustment);
  }

2. computeFinalEntryScoreはuncapped値にapplyFinalScoreCapsを適用する。

  function computeFinalEntryScore(symbolScore: number, input: ScoringInput, riskScore: number): number {
    return applyFinalScoreCaps(computeFinalEntryScoreUncapped(symbolScore, input), input, riskScore);
  }

3. marketScoreAdjustment / sectorScoreAdjustment を小関数化する。
4. 既存thresholdやcap定数は変更しない。

受け入れ条件:
- neutral fixtureでfinalEntryScoreが45-55。
- weak fixtureがentry thresholdを超えない。
- bullish market/sectorで軽く加点される。
- bearish market/sectorで減点される。
- forbidden/data/event/bearish capsは維持。

追加/更新テスト:
- neutral components stay neutral
- weak symbol does not become entry candidate
- bullish market adjusts but does not dominate
- bearish market cap applies


P0-2: LLM補正後にも全final score capを適用
----------------------------------------

対象ファイル:
- src/lib/scoring/scoring-engine.ts
- src/lib/analysis/llm-enhancer.ts
- src/lib/scoring/scoring.test.ts
- src/lib/analysis/analysis.test.ts

現在の問題:
applyLlmAdjustmentがriskOnlyFinalCapのみを使っており、low data, event blocker, bearish market, forbidden capを再適用しない可能性がある。

変更内容:
1. applyLlmAdjustmentのsignatureを変更する。

  export function applyLlmAdjustment(
    scores: ScoringOutput,
    adj: LlmScoreAdjustment,
    input: ScoringInput,
  ): ScoringOutput

2. LLM補正後のsymbolScoreを使い、finalEntryScoreを以下で再計算する。

  u.finalEntryScore = computeFinalEntryScore(symbolScore, input, u.riskScore);

3. src/lib/analysis/llm-enhancer.tsの呼び出し元を修正する。

  const scores = applyLlmAdjustment(logic.scores, adjustment, logic.scoringInput);

4. riskOnlyFinalCapを削除する。

受け入れ条件:
- low data時、LLM最大加点後もlowDataConfidence cap以下。
- event blocker時、LLM最大加点後もeventBlocker cap以下。
- bearish market時、LLM最大加点後もbearishMarket cap以下。
- forbidden時、LLM最大加点後もforbidden cap以下。

追加/更新テスト:
- optimistic LLM adjustment cannot bypass low data cap
- optimistic LLM adjustment cannot bypass event blocker cap
- optimistic LLM adjustment cannot bypass bearish market cap
- optimistic LLM adjustment cannot bypass forbidden cap


P1-1: ChartContainerをrender-props化しResponsiveContainerを削除
-------------------------------------------------------------

対象ファイル:
- src/components/ui/chart.tsx
- src/components/candidate-visuals.tsx
- src/components/fundamental-visuals.tsx

現在の問題:
ChartContainerはResizeObserverでready判定しているが、width/heightを子に渡していない。利用側にはResponsiveContainerが残っている。

変更内容:
1. ChartContainer APIを以下に変更する。

  type ChartSize = { width: number; height: number };

  export function ChartContainer({
    className,
    children,
  }: {
    className?: string;
    children: (size: ChartSize) => React.ReactNode;
  })

2. ResizeObserverでcontentRect.width/heightをstateに保存する。
3. width <= 0 または height <= 0 の間はplaceholderまたはnullを返す。
4. candidate-visuals.tsxからResponsiveContainer importを削除。
5. fundamental-visuals.tsxからResponsiveContainer importを削除。
6. Rechartsの各chartへ width={width} height={height} を渡す。

受け入れ条件:
- ResponsiveContainer importが0件。
- chartが描画される。
- resize時にchartが追従する。
- width/height=0でクラッシュしない。

追加/更新テスト:
- ResizeObserver mock
- measured size passed to children
- zero size does not crash

P1-3: LLM settings入力検証強化
-----------------------------

対象ファイル:
- src/app/api/settings/llm/route.ts
- src/lib/llm/settings.ts

現在の問題:
provider, baseUrl, temperature, costなどの検証が弱い。

変更内容:
- providerはdeepseek/openai-compatibleのみ。

- temperatureは0以上1.5以下。

受け入れ条件:
- invalid provider -> 400
- invalid temperature -> 400
- localhost baseUrl -> 400
- private IP baseUrl -> 400
- valid input -> save succeeds

追加/更新テスト:
- invalid provider
- invalid baseUrl
- negative cost
- too high temperature
- empty model
