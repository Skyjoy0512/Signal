09 Test Plan Improvements
=========================

目的
----

投資分析ロジックは、小さな式変更でも候補判定やUI表示が大きく変わる。
そのため、unit test、fixture regression、API test、UI chart test、LLM integration testを追加し、スコアリング変更時の事故を防ぐ。


TEST-1: P0 scoring regression tests
----------------------------------

対象:
- src/lib/scoring/scoring.test.ts

追加ケース:

1. neutral components stay neutral
- opportunity = 50
- entryTiming = 50
- risk = 50
- conviction = 50
- market neutral
- sector neutral
期待:
- finalEntryScoreは45-55。

2. weak symbol does not become entry candidate
- symbolScore相当30
- neutral market/sector
期待:
- finalEntryScoreはentry threshold未満。

3. bullish market adjusts but does not dominate
- symbolScore中立
- bullish market/sector
期待:
- 加点はされるが70を超えない。

4. bearish market cap applies
期待:
- finalEntryScore <= FINAL_SCORE_CAPS.bearishMarket

5. low data cap applies
期待:
- finalEntryScore <= FINAL_SCORE_CAPS.lowDataConfidence

6. event blocker cap applies
期待:
- finalEntryScore <= FINAL_SCORE_CAPS.eventBlocker

7. forbidden cap applies
期待:
- finalEntryScore <= FINAL_SCORE_CAPS.forbidden


TEST-2: LLM adjustment guard tests
---------------------------------

対象:
- src/lib/analysis/analysis.test.ts
- src/lib/scoring/scoring.test.ts

追加ケース:

1. optimistic LLM adjustment cannot bypass low data cap
入力:
- dataConfidence < 60
- LLM score_adjustment +10
期待:
- finalEntryScore <= lowDataConfidence cap

2. optimistic LLM adjustment cannot bypass event blocker
入力:
- eventBlockerActive = true
期待:
- finalEntryScore <= eventBlocker cap

3. optimistic LLM adjustment cannot bypass bearish market cap
入力:
- market.condition = bearish
期待:
- finalEntryScore <= bearishMarket cap

4. optimistic LLM adjustment cannot bypass forbidden cap
入力:
- isForbidden = true
期待:
- finalEntryScore <= forbidden cap

5. critic downgrade blocks positive adjustment
入力:
- critic.required_downgrade = true
期待:
- positive adjustment is ignored or reduced

6. malformed LLM JSON repair
入力:
- broken JSON
期待:
- repair成功ならvalid output
- repair失敗ならfallback to rule-based
