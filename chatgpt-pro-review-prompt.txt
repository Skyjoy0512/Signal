Repository:
https://github.com/Skyjoy0512/Signal

プロジェクト概要:
このプロジェクトは、個人投資家向けの日本株投資判断支援アプリ「Signal」です。
目的は、銘柄データ、テクニカル指標、市場/セクター/テーマ/銘柄レイヤー分析、スコアリング、LLM補助分析を組み合わせて、投資候補の発見と判断材料の整理を行うことです。

重要:
これは投資助言の依頼ではありません。
ソフトウェア設計、分析ロジック、データ設計、保守性、テスト容易性、将来のLLM差し替え容易性のレビューです。

重点的に見てほしい領域:
- src/lib/analysis
- src/lib/scoring
- src/lib/intelligence
- src/lib/indicators
- src/lib/llm
- src/lib/jobs/daily-scan.ts
- src/lib/mock/provider.ts
- supabase/migrations
- docs

出力形式:
まずレビュー結果を簡潔に出してください。
その後、最後に必ず以下の形式で「Codexに渡す実装指示書」を出力してください。

最終セクションのタイトルは必ず以下にしてください。

# CODEX_IMPLEMENTATION_INSTRUCTIONS.txt

このセクションは、私がそのままコピーしてCodexに渡します。
したがって、Codexが迷わず実装できるように、具体的なファイル名、関数名、変更内容、テスト条件を含めてください。

実装指示書には以下を含めてください。

1. Objective
- 今回の実装目的を1段落で説明

2. Scope
- 対象ファイル/ディレクトリ
- 対象外にするもの

3. Priority
P0/P1/P2/P3で分類してください。
P0とP1は必ず実装可能な粒度に分解してください。
P2/P3は将来タスクとして残してもよいです。

4. Implementation Tasks
各タスクごとに以下を含めてください。
- Task ID
- Priority
- Target files
- Current problem
- Required change
- Detailed implementation steps
- Acceptance criteria
- Tests to add/update
- Risk/notes

5. Suggested File Structure
必要なら新規ファイル/移動/分割案を示してください。

6. Testing Plan
- npm run typecheck
- npm test
- npm run lint
に加えて、追加すべきテストがあれば書いてください。

7. Non-goals
今回やらないことを明記してください。
特にUIデザイン変更や投資助言の作成は対象外にしてください。

8. Codex Notes
Codexへの注意事項を書いてください。
例:
- 既存のUI作業と衝突しないよう、主に src/lib 配下に限定する
- 秘密情報をGitに入れない
- 既存テストを壊さない
- 変更後は typecheck/test/lint を実行する
- 可能なら小さなコミット単位に分ける

重要な制約:
- 一般論ではなく、このRepositoryの実コードに基づいて指摘してください。
- 具体的なファイルパスと関数名を挙げてください。
- 改善案だけでなく、実装可能な作業単位まで落としてください。
- 実装指示書はCodexが読んでそのまま作業に入れる詳細度にしてください。
- 日本語で出力してください。
